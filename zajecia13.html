Section1: Zapytania o media

Zapytania o media - umożliwia tworzenie nowoczesnych responsywnych stron internetowych, które wyglądają równie dobrze na każdym ekranie, czy to na komputerze stacjonarnym, czy smartfonie. Wszystko sprowadza się do zastosowania jednej lub drugiej reguły CSS, w zależności od wielkości okna przeglądarki (rzutni), ustawień przeglądarki lub możliwości urządzenia.

Algorytm stosowania stylów jest następujący:

  • Deweloper opisuje zestaw zapytań o media i zawarte w nich reguły CSS.
  • Przeglądarka śledzi zmiany rozmiaru rzutni.
  • Przeglądarka stosuje reguły CSS z zapytań o media, które pasują do bieżącego rozmiaru rzutni.

Oznacza to, że zapytania o media to instrukcje w postaci: „gdy strona została otwarta na pulpicie, chcę, aby kolor tekstu był czerwony, a po otwarciu w telefonie - zielony”.

Section2: Składnia

            @media media-type and (media-feature) {
            /*
            * Zestaw reguł CSS do zastosowania w dokumencie,
            * jeśli warunek sprawdzenia typu media i wyrażenia jest prawdziwy
            */
            }

Zapytanie o media to specjalna konstrukcja CSS, zadeklarowana przez dyrektywę @media, po której może znajdować się typ mediów (media-type) i cecha mediów (media-feature) - wyrażenia testujące właściwości urządzenia (na przykład szerokość rzutni). Cecha mediów jest wyrażeniem logicznym, którе zwraca prawdę lub fałsz.

Na przykład jeśli chcesz, aby tło "body" było pomarańczowe, gdy szerokości rzutni 900px i szersza, trzeba napisać następujące zapytanie o media. 

            @media screen and (min-width: 900px) {
            body {
            background-color: orange;
                }
            }

Mówiąc prostymi słowami można to odczytać jako: „jeśli strona jest wyświetlana na ekranie o szerokości co najmniej 900 pikseli, należy zastosować określoną regułę CSS”.

Warunek sprawdzania może być tak złożony lub tak prosty, jak wymaga tego sytuacja. W większości przypadków wystarczy określić typ mediów urządzenia (najczęściej screen), i sprawdzić tylko szerokość rzutni. 

            /*
            To zapytanie o media zostanie zastosowane podczas drukowania dokumentu.
            */
            @media print {
                /* ... */
            }
            
            /*
            To zapytanie o media zostanie zastosowane, gdy rzutnia będzie szersza niż 400 pikseli.
            */
            @media screen and (min-width: 400px) {
                /* ... */
            }

            /*
            Typ screen można pominąć, jeśli style są pisane tylko dla ekranów.
            */
            @media (min-width: 400px) {
                /* ... */
            }

Section3: Typy mediów

Typ mediów służy do opisywania typów urządzeń, na których może być wyświetlana strona internetowa. W sumie są trzy typy obsługiwane przez wszystkie nowoczesne przeglądarki.

  • all - jeśli nie określisz typu nośnika, domyślnie będzie używana ta wartość, która oznacza dowolne urządzenie.
  • print - pasuje do drukarek i urządzeń przeznaczonych dla odtwarzania wersji drukowanej, na przykład przeglądarka internetowa wyświetlająca dokument w trybie podglądu.
  • screen - opisuje urządzenia z fizycznym ekranem: smartfony, tablety, monitory, telewizory itp. czyli wszystko co nie obejmuje typu print

Section4: Cechy mediów

Dwie najczęściej używane cechy mediów, które pozwalają na zdefiniowanie szerokości rzutni przeglądarki, to min-width i max-width. Określa się minimalna (min-width) lub maksymalna (max-width) dopuszczalna szerokość rzutni, przy której są stosowane reguły z zapytań o media. 

            * Będzie używany, gdy szerokość widocznego obszaru (rzutni) jest większa niż 900 pikseli */
            @media (min-width: 900px) {
                body {
                    background-color: green;
                }
            }

            /* Będzie używany, gdy szerokość widocznego obszaru jest mniejsza niż 600 pikseli */
            @media (max-width: 600px) {
                body {
                    background-color: yellow;
                }
            }

Jeśli reprezentujemy szerokość rzutni jako linię prostą od 0 do nieskończoności, to opisujemy przerwę (odstęp), w którym konieczne jest zastosowanie jakichś stylów.

Rozmiar rzutni, w którym cały układ lub jego poszczególne części zmieniają swoje style, czyli zmienia się projekt (design) nazywa się punktem przerwania (breakpoint).

Przy zmianie szerokości rzutni w przykładzie, przed 600px tło "body" będzie żółte, a po 900px zielone. W zakresie od 601 do 899 pikseli kolor jest biały, wartość domyślna, ponieważ żadne zapytanie o media nie będzie pasować. 

            body {
                background-color: #fff;
            }

            @media screen and (max-width: 600px) {
                body {
                    background-color: #ffc107;
                }
            }

            @media screen and (min-width: 900px) {
                body {
                    background-color: #388e3c;
                }
            }

Jak opisać przerwę, na przykład od 600px do 900px? W przypadku złożonych cech media należy używać operatorów logicznych.

Section5: Operatory logiczne

Typ i cechy mediów można oddzielić opcjonalnymi operatorami logicznymi not, and i only - wartość domyślna. Pełna składnia zapytania o media jest następująca.

Section6: Operator and

Operator and (dosłownie "i") jest używany nie tylko między typem nośnika a warunkiem mediów, ale także do łączenia wielu cech media z kontrolami wartości. 

            @media screen and (min-width: 400px) and (max-width: 800px) {
                body {
                    background-color: red;
                }
            }

Takie zapytanie o media zostanie wykonane tylko wtedy, gdy strona internetowa jest otwarta na ekranie, a szerokość rzutni mieści się w zakresie od 400px do 800px.

Section7: Operator ,

Operator , (dosłownie "albo") pozwala określić zestaw wyrażeń, gdy przy wykonaniu przynajmniej jednego z nich zostanie wykonane zapytanie o media.

Na przykład musisz zastosować style do 600px lub po 900px.

 @media (max-width: 600px), (min-width: 900px) {
     body {
         background-color: orange;
     }
 }

Oznacza to, że jest to po prostu wyliczenie zestawu zapytań o media, do których należy zastosować style. Można to zapisać jako dwa różne zapytania o media. Ale wtedy kod stylu "body" zostanie zduplikowany. 

            @media (max-width: 600px) {
              body {
                background-color: orange;
              }
            }

            @media (min-width: 900px) {
              body {
                background-color: orange;
              }
            }

W przyszłych wersjach CSS, operator , zostanie zastąpiony przez operator or.

Section8: Operator not

Operator not (dosłownie "nie") umożliwia wykonanie negacji, czyli anulowanie zapytania o media. Słowo kluczowe not jest dołączane na początku zapytania o media i dotyczy całego zapytania. Jest używany bardzo rzadko, w niektórych szczególnych przypadkach.

Na przykład napiszmy warunek, w którym style powinny być stosowane wszędzie z wyjątkiem drukowania. 

            @media not print {
              /* ... */
            }

Używając operatora not należy określić typ nośnika, ponieważ domyślnie zostanie ustawiony na all a wyrażenie not all będzie dosłownie odczytywane jak „nie wszystkie”, a zapytanie o media nigdy nie zostało wykonane. 

            /* Nigdy nie zostanie wykonany */
            @media not (max-width: 500px) {
              /* ... */
            }

            /* Zostanie wykonany, jeśli szerokość będzie większa niż 500 pikseli. */
            @media not screen and (max-width: 500px) {
              body {
                background-color: tomato;
              }
            }

            /*
            W tym przypadku, aby ułatwić odczytanie kodu,
            lepiej używać min-width: 500px.
            */
            @media (min-width: 500px) {
              body {
                background-color: tomato;
              }
            }

Section9: Nadpisywanie stylów

Nic nie stoi na przeszkodzie, aby przeglądarka mogła zastosować więcej niż jedno zapytanie o media w tym samym czasie, jeśli spełniają one warunek. Zapytania o media nie dodają specyficzności do selektorów, które zawierają, ale kolejność reguł nadal ma znaczenie. Oznacza to, że reguły występujące w zapytaniach o media uczestniczą w standardowym kaskadowaniu podczas tworzenia ostatecznych stylów elementu. 

              /* Podstawowy styl */
              body {
                background-color: #fff;
              }

              /* Przy 900px i więcej nadpisujemy tło  */
              @media screen and (min-width: 900px) {
                body {
                  background-color: #388e3c;
                }
              }

W narzędziach programistycznych możesz zobaczyć, jak stosowane są reguły CSS, a kolor tła elementu "body" jest nadpisywany.

W takim przypadku nadpisywane są tylko jednakowe właściwości, a nie cała reguła. Dzięki temu możesz zrobić ciasto warstwowe ze stylów bez powielania poprzednich, które pasują również do aktualnego zapytania o media. 

              /* Base styles */
              body {
                color: #2a2a2a;
                background-color: #fff;
                font-size: 16px;
                line-height: 1.5;
                font-family: sans-serif;
              }

              /* From 600px and wider, override the background, font and color. */
              @media (min-width: 600px) {
                body {
                  background-color: #2196f3;
                  font-size: 20px;
                  color: #fff;
                }
              }

              /* From 900px and wider, override the background. */
              @media (min-width: 900px) {
                body {
                  background-color: #4caf50;
                }
              }

Tak będą wyglądać style elementu "body" w narzędziach programistycznych, gdy szerokość rzutni jest większa niż 900px.

Section10: Metaznacznik viewport

Rzutnia (viewport) - to widoczny, prostokątny obszar strony internetowej, który nie obejmuje paska adresu, paska zakładek i innych elementów usługowych przeglądarki.

Przeglądarki mobilne renderują stronę internetową w widocznym obszarze, który jest szerszy niż fizyczny ekran urządzenia. Domyślnie szerokość obszaru (rzutni) w przeglądarkach mobilnych zależy od producenta urządzenia, nie odpowiada rozmiarowi ekranu urządzenia i najczęściej wynosi 980px. Oznacza to, że przeglądarka mobilna traktuje ten rozmiar jako rozmiar, względem którego powinna stosować zapytania o media.

Oznacza to, że zapytania o media przeznaczone do urządzeń mobilnych po prostu nie zostaną zastosowane lub zostaną ponownie nadpisane w przeglądarce mobilnej. Na przykład podczas otwierania strony internetowej na ekranie o szerokości 400px, zostanie zastosowane zapytanie o media z min-width: 768px nadpisując style mobilne i zobaczymy wersję strony przeznaczoną dla tabletów, chociaż ona została otwarta w telefonie.

Strony przystosowane do przeglądania na różnych urządzeniach muszą zawierać w sekcji "head" metaznacznik viewport. On informuje przeglądarkę, jak kontrolować rozmiar i skalę rzutni. 

              /* Inne metaznaczniki */
              meta name="viewport" content="width=device-width, initial-scale=1.0" />
              title>
              Metaznacznik viewport jest ważny w przypadku responsywnych stron
              /title>
              /head>
  • width=device-width - ustawia szerokość widocznego obszaru (width), która jest równa fizycznej szerokości ekranu urządzenia (device-width).
  • initial-scale=1.0 - ustawia stosunek (skalę) 1:1 między pikselami CSS a pikselami sprzętowymi urządzenia

Section11: Narzędzia deweloperskie

Aby przejść do trybu responsywnego w przeglądarce Chrome, musisz otworzyć narzędzia programistyczne i kliknąć ikonę 1. Pojawią się dodatkowe narzędzia do pracy z responsywnym układem. Aby wyjść z tego trybu, musisz ponownie kliknąć ikonę 1.

Aby zmienić szerokość rzutni przez proste rozciągnięcie, użyj elementu 2. Efekt jest taki, jakby okno przeglądarki zostało rozciągnięte lub "skurczone", zmieniając w ten sposób szerokość rzutni.

W górnej części narzędzi znajduje się zestaw dodatkowych elementów sterujących.

  • 3 - wybierz urządzenie o precyzyjnych wymiarach, takie jak iPhone, iPad, Pixel itd. lub wartość domyślną - Reponsive, tryb responsywny z dowolną szerokością rzutni.
  • 4 - aktualne wymiary szerokości i wysokości rzutni. Możesz wprowadzić wartości ręcznie.
  • 5 - zmiana skali, w przypadku gdy praca przebiega z dużymi rozmiarami i trzeba spojrzeć na cały układ jako całość.
  • 6 - możliwość symulacji obrotu urządzenia w trybie pionowym lub poziomym.

Nieco poniżej zestawu dodatkowych elementów sterowania znajduje się panel do szybkiej zmiany szerokości rzutni za pomocą standardowych punktów przerwania.

Section12: Typy układów

Początkowo strony internetowe miały stały układ. Jeśli pojemnik miał szerokość 1200 pikseli, pozostawał szeroki na 1200 pikseli, bez względu na to, na jakim ekranie go otworzysz, dużym czy małym. W dzisiejszej różnorodności urządzeń mobilnych to podejście jest przestarzałe.

Nowoczesna sieć internetowa przyjmuje nowe podejścia do tworzenia witryn internetowych. Łączą one elementy układu płynnego z nowymi, nowoczesnymi technikami, takimi jak adaptacyjny (adaptive) i responsywny (responsive) układ. Techniki „adaptacyjnego” i „responsywnego” układu różnią się ustawieniem szerokości kontenera i zagnieżdżonych elementów.

Responsywna (responsive) strona internetowa - posiada kilka opcji wyświetlania, przejścia między nimi są płynne, elementy i pojemnik rozciągają się jak guma. Podczas zmiany rozmiaru rzutni bloki płynnie się kurczą lub rozciągają, a gdy pojawia się punkt przerwania, (breakpoint) - zmieniają swoje położenie tak, aby optymalnie zajmować całą wolną przestrzeń poziomą.

Adaptacyjna (adaptive) strona internetowa - ma kilka opcji wyświetlania, w przeciwieństwie do strony responsywnej - dosłownie. Projekt zmienia się w szarpnięciach w sztywno określonych punktach przerwania i nie rozciąga się między nimi.

Podczas składania zapytań o media należy skupić się na punktach przerwania projektu, czyli wartościach szerokości rzutni, w których projekt znacznie się zmienia. Możesz je zdefiniować zgodnie z układem narysowanym przez projektanta. 

            /* Common container styles. */
            .container {
              min-width: 320px;
              margin-left: auto;
              margin-right: auto;
              background-color: #ffc107;

              /*  Text styles  */
              text-align: center;
              line-height: 180px;
              font-size: 32px;
              white-space: nowrap;
            }

            /* Set the maximum width of the responsive container. */
            .container.responsive {
              max-width: 1140px;
              margin-bottom: 40px;
            }

            /* Set the base width of a adaptive container. */
            .container.adaptive {
              max-width: 400px;
            }

            @media screen and (min-width: 600px) {
              .container {
                background-color: #00bcd4;
              }

              /* Change the current width of the adaptive container at each breakpoint. */
              .container.adaptive {
                max-width: 600px;
              }
            }

            @media screen and (min-width: 900px) {
              .container {
                background-color: #8bc34a;
              }

              /* Change the current width of the adaptive container at each breakpoint. */
              .container.adaptive {
                max-width: 900px;
              }
            }

            @media screen and (min-width: 1140px) {
              .container {
                background-color: #ff5252;
              }

              /* Change the current width of the adaptive container at each breakpoint. */
              .container.adaptive {
                max-width: 1140px;
              }
            }

            .container .label {
              margin-top: 0;
            }

Dla obu kontenerów ustawiana jest właściwość min-width - szerokość rzutni, poniżej której projekt nie jest obsługiwany. Użytkownik z węższym ekranem będzie miał poziomy pasek przewijania. 

            .container { 
              min-width: 320px;
            }

Kontener responsywny jest ustawiany raz na właściwość max-width w stylach podstawowych, tak aby rozciągał się dowolnie, ale maksymalnie nie był szerszy niż określona wartość. 

            .container.responsive {
              max-width: 1140px;
            }

Kontener adaptacyjny otrzymuje początkową wartość maksymalnej szerokości, po czym jest ponownie definiowany w każdym punkcie przerwania. 

            /* Ustawiamy szerokość podstawową adaptacyjnego kontenera  */
            .container.adaptive {
              max-width: 320px;
            }

            @media screen and (min-width: 600px) {
              .container.adaptive {
                max-width: 600px;
              }
            }

            @media screen and (min-width: 900px) {
              .container.adaptive {
                max-width: 900px;
              }
            }

            @media screen and (min-width: 1140px) {
              .container.adaptive {
                max-width: 1140px;
              }
            }

Które podejście należy zastosować? Wszystko zależy od projektu, rodzaju strony internetowej i możliwości finansowych klienta. W przypadku większości witryn dla małych i średnich przedsiębiorstw wersja adaptacyjna jest wystarczająca. Układ responsywny jest droższy w projektowaniu i rozwoju, ponieważ jego wykonanie jest trudniejsze i dłuższe, ale to jest niezastąpione w interfejsach nowoczesnych aplikacji internetowych. Czasami zadania biznesowe można lepiej wykonać za pomocą oddzielnej wersji telefonu komórkowego lub nawet natywnej aplikacji zamiast strony internetowej.

Section13: Strategia Mobile First

Historycznie rzecz biorąc, projektanci zaczęli opracowywać projekt witryny dla dużego ekranu, to znaczy, że pierwszy i „główny” projekt został wykonany dla komputera stacjonarnego, ponieważ oznacza to dużą funkcjonalność.

Problem z tym podejściem polega na tym, że pierwszeństwo mają użytkownicy komputerów i laptopów, chociaż obecnie, według statystyk, dominuje ruch mobilny. Wiele elementów projektu i funkcjonalności, które świetnie wyglądają i działają na pulpicie komputera, trudno przenieść na ekran telefonu komórkowego.

Strategia Mobile First zaczyna się odwrotnie - najpierw powstaje wersja mobilna, a następnie dla tabletów i wersja desktopowa. Dzięki temu aktualne informacje są łatwo dostępne i można ukrywać lub zmieniać elementy na dowolnym ekranie.

Strona internetowa musi być dobrze zoptymalizowana pod kątem wyszukiwarek (SEO) i spełniać wszystkie wymagania UX (user experience), tak aby wejście użytkownika na stronę było jak najbardziej wygodne i zrozumiałe za pośrednictwem jego urządzenia mobilnego. Dlatego jednym z najważniejszych wymagań w rozwoju Mobile First jest:

  • Najpierw pokazać najważniejsze treści.
  • Witryna powinna być lekka i zoptymalizowana, ponieważ prędkość połączenia z siecią komórkową może być niska w zależności od lokalizacji użytkownika.
  • Witryna internetowa nie powinna ładować więcej zasobów, niż potrzebuje użytkownik, aby uzyskać potrzebne informacje.
  • Dodatkowe informacje powinny być ładowane tylko na żądanie użytkownika.

Takie podejście przynosi ogromne korzyści podczas projektowania strony internetowej.

  • Jedna strona internetowa - tylko jeden projekt na wszystkie urządzenia. Zmniejsza liczbę wymaganych programistów.
  • Wygodny interfejs - użytkownicy najpierw otrzymają ważną zawartość strony.
  • Szybkość ładowania - strona będzie ładowała się szybciej na urządzeniach mobilnych ze względu na mniejszą liczbę pobieranych i wyświetlanych zasobów.
  • Ranking wyszukiwania - na przykład Google nie tylko nadaje priorytet witrynom zoptymalizowanym pod kątem urządzeń mobilnych, ale także bierze pod uwagę czas otwarcia strony.

Section14: Mobile-first CSS

Technicznie implementacja Mobile First jest dość prosta - style dla urządzeń mobilnych to podstawowe style poza zapytaniami o media, po których do każdego punktu przerwania dodawane jest zapytanie o media, w którym są nadpisywane niezbędne podstawowe style, style z poprzedniego zakresu lub nowe. Dlatego w zapytaniach o media używają się głównie cechy media min-width. 

            .element {
              /* Style podstawowe */
            }

            @media screen and (min-width: szerokość-tabletu) {
              .element {
                /* Style tabletu */
              }
            }

            @media screen and (min-width: szerokość-desktopu) {
              .element {
                /* Style desktopu */
              }
            }

Tworzy to lekką wersję stylów dla urządzeń mobilnych i wszystkie style dla szerszych ekranów są zamykane wewnątrz odpowiednich zapytań o media. Nie zmienia to znaczników HTML.

Zaletą tego podejścia jest to, że w przypadku stylów mobilnych prawie nigdy nie trzeba określać pozycji, na przykład przy użyciu Flexbox, ponieważ wszystkie treści idą w jednej kolumnie po drugiej - domyślne zachowanie elementów blokowych w standardowym przepływie dokumentów. Oznacza to, że w zapytaniach o media dla szerszych ekranów nie musisz na nowo definiować pozycjonowania.

Poniższa ilustracja przedstawia strukturę zapytań o media i stylów dla prostej strony z 2 punktami przerwania 768px i 1024px. Reguły CSS w zapytaniach o media po prostu dodają nowe style lub nadpisują poprzednie.

Przejdź do pełnej wersji przykładowego edytora pod linkiem i zmień szerokość obszaru z wynikiem. Uwaga, początkowo nie ma nawet @media, ponieważ style są pisane dla wąskiego ekranu jako podstawa. Później, podczas pisania stylów na szersze ekrany, dodajemy zapytania o media.

Zalety takiego układu:

  • Podstawowy kod CSS jest bardziej czysty, ponieważ przepływ dokumentu i model blokowy pomagają nam w pozycjonowaniu elementów interfejsu użytkownika.
  • Dziedziczenie stylów od wąskich do szerokich ekranów.
  • Minimalne przedefiniowanie stylów pozycjonowania wynikające z pierwszego punktu.

Section15: Cecha mediów max-width

W większości sytuacji wystarczy użyć min-width i dodać lub nadpisać style. Są jednak sytuacje, w których użycie cech mediów max-width sprawia, że kod jest czystszy i bardziej czytelny.

Wyobraźmy sobie taką sytuację - w ostatnim przykładzie elementy muszą dodać dolny margines tylko na urządzeniu mobilnym, czyli do 767px (style tabletu zaczynają się od 768). A dla odstępu w przypadku tabletu, od 768px do 1023px, elementy muszą mieć obramowanie.

Użycie tylko min-width spowoduje, że style zapytań o media z min-width: 768px będą musiały wyzerować dolny margines elementów. W stylach zapytania o media z min-width: 1024px wyzerujemy style obramowania. Nie jest to najlepsze rozwiązanie. 

            ŹLE:
            .box {
              margin-bottom: 10px;
            }

            @media (min-width: 768px) {
              .box {
                margin-bottom: 0;
                border: 1px solid black;
              }
            }

            @media (min-width: 1024px) {
              .box {
                border: none;
              }
            }
          
            DOBRZE:
            @media (max-width: 767px) {
              .box {
                margin-bottom: 10px;
              }
            }

            @media (min-width: 768px) and (max-width: 1023px) {
              .box {
                border: 1px solid black;
              }
            }

W takich sytuacjach dobrą praktyką jest zamykanie określonych stylów w pewnych przerwach, a cecha mediów max-width nam w tym pomoże.
